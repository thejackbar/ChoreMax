Top 5 Critical Issues

  • CriticalDouble-Slash Canonical URLs on ALL 50 Pages
    Every page has a canonical like https://maxgeo.com//exploration instead of https://maxgeo.com/exploration. This is caused by the Next.js metadataBase having a trailing slash. Google may treat these as different URLs, splitting link equity and causing indexing confusion across the entire site.
  • Critical5 Dead URLs (404) in Sitemap
    Sitemap includes 5 URLs returning 404: /datashed-training, /assay-management, /drilling-management, /logchief-training, /logchief-lite-training. These waste crawl budget and signal poor site maintenance to search engines.
  • Critical24 of 50 Pages (48%) Missing Meta Descriptions
    Homepage, /exploration, /grade-control, /training, /legal, /software/datashed, /software/logchief, 1 insight article, and all 18 event pages have no meta description. Google will auto-generate snippets that may not represent your content well.
  • CriticalHero Image loading="lazy" Kills LCP
    The first visible image (hero: 1920x1080 JPG) has loading="lazy", which delays the Largest Contentful Paint. The hero image should use loading="eager" or priority in Next.js Image component.
  • CriticalHSTS max-age=0 (Security Effectively Disabled)
    The Strict-Transport-Security header has max-age=0, which effectively disables HSTS protection. Should be at least max-age=31536000 (1 year). Also missing: Content-Security-Policy, X-Frame-Options, Permissions-Policy, Referrer-Policy.

Top 5 Quick Wins

  • QuickFix metadataBase Trailing Slash
    Change metadataBase: new URL('https://maxgeo.com/') to metadataBase: new URL('https://maxgeo.com') in your root layout. This single change fixes all 50 double-slash canonicals instantly.
  • QuickRemove 5 Dead URLs from Sitemap
    Delete the 5 pages returning 404 from the sitemap generation logic. If using next-sitemap or a custom sitemap route, exclude these paths.
  • QuickAdd priority={true} to Hero Image
    In the homepage hero component, add priority prop to the Next.js Image component. This removes loading="lazy" and adds fetchpriority="high", fixing LCP.
  • QuickAdd 3 Software Pages to Sitemap
    /software/datashed, /software/logchief, and /software/logchief-lite all return 200 but are missing from the sitemap. Add them.
  • QuickAdd Meta Descriptions to Core Pages
    The homepage, /exploration, /grade-control, and /training are high-value pages missing descriptions. Add unique, keyword-rich descriptions in the Sanity CMS metadata fields.

5. Performance (Core Web Vitals) 50/100

LCP Analysis

  • CriticalHero Image: loading="lazy" on LCP Element
    The hero image (1920x1080 JPG, ~90 KB) uses loading="lazy" which delays browser fetching until the image is near viewport. For the LCP element, this can add 500-2000ms to LCP. Fix: add priority prop to Next.js Image component.
  • HighImage Format: JPEG Only (No WebP/AVIF)
    Next.js Image optimization endpoint returns image/jpeg (content-type confirmed). On Cloudflare, the built-in image optimizer may not be configured. The response headers show no Accept negotiation for modern formats. Configure Cloudflare Image Resizing or use Sanity's image transform API with format=webp.
